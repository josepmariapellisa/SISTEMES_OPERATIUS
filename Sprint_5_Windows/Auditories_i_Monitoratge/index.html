    Introducció a les Auditories i al Monitoratge en Windows

    En sistemes operatius Windows, especialment en entorns corporatius o crítics, és fonamental disposar de mecanismes que permetin supervisar, registrar i analitzar les activitats dels usuaris i del sistema. Aquestes funcionalitats són essencials per garantir la seguretat, el rendiment i el compliment normatiu.

    Aquest document presenta una introducció detallada a dues pràctiques fonamentals: les auditories i el monitoratge dins de Windows.

    1. Auditories en Windows

    1.1 Què és una auditoria?

    Una auditoria és el procés de registrar i examinar els esdeveniments i accions que ocorren dins del sistema operatiu. A Windows, aquest procés es realitza mitjançant el subsistema d'auditoria del sistema de seguretat, que registra informació detallada sobre accions específiques dels usuaris o del sistema.

    1.2 Objectius de l’auditoria

    • Detecció d’activitats malicioses (p. ex. intents de connexió no autoritzats).
    • Seguiment de l’accés a dades sensibles.
    • Verificació del compliment de polítiques internes o normatives (com ISO 27001, GDPR, etc.).
    • Anàlisi postmortem després d’un incident de seguretat.

    1.3 Tipus d’esdeveniments auditables

    Windows permet auditar diversos tipus d’esdeveniments, com ara:

    • Inici i tancament de sessió.
    • Canvis en comptes d’usuari i grups.
    • Accés a fitxers o carpetes específiques.
    • Ús de privilegis especials (p. ex. elevació de permisos).
    • Modificacions en polítiques de seguretat.

    1.4 Configuració d’auditories

    Per configurar una auditoria, es poden utilitzar les eines següents:

    a) Polítiques locals de seguretat (secpol.msc)

    1. Obre el menú Inici i escriu secpol.msc.
    2. Navega a: Polítiques locals > Política d’auditoria.
    3. Activa les categories d’auditoria (p. ex. "Auditar esdeveniments de connexió").

    b) Polítiques de grup (GPO)

    1. Obre gpedit.msc o configura les GPOs des del controlador de domini.
    2. Ruta: Configuració de l’equip > Configuració de Windows > Configuració de seguretat > Polítiques locals > Política d'auditoria.
    3. També es pot utilitzar la configuració avançada d’auditoria (Audit Policy (Advanced)).

    1.5 Revisió dels esdeveniments

    Els esdeveniments auditats es poden consultar mitjançant:

    • Visor d'esdeveniments (eventvwr.msc)
    • Ruta: Registres de Windows > Seguretat
    • Cada entrada inclou: ID de l'esdeveniment, usuari, hora, i detall de l'acció realitzada.

    1.6 Exemples pràctics

    • Esdeveniment 4624: Inici de sessió exitós.
    • Esdeveniment 4625: Inici de sessió fallit.
    • Esdeveniment 4663: Accés a un objecte (p. ex. fitxer o carpeta).
    • Esdeveniment 4720: Creació d’un compte d’usuari.

    2. Monitoratge en Windows

    2.1 Què és el monitoratge?

    El monitoratge és la pràctica de supervisar l’activitat i rendiment del sistema operatiu, els serveis i les aplicacions en temps real o de forma retrospectiva. Serveix per detectar problemes, optimitzar recursos i garantir una operació estable.

    2.2 Objectius del monitoratge

    • Assegurar la disponibilitat dels serveis.
    • Identificar colls d’ampolla en el rendiment.
    • Detectar fallades de maquinari o programari.
    • Prevenir atacs per comportament anòmal (p. ex. ús excessiu de CPU).

    2.3 Eines natives de monitoratge

    a) Gestor de tasques (Task Manager)

    • Monitoratge bàsic de:
    • CPU, memòria, disc i xarxa.
    • Processos actius.
    • Serveis en execució.

    b) Monitor de rendiment (perfmon.msc)

    • Permet crear jocs de recopilació de dades.
    • Supervisió personalitzada d’indicadors com:
    • % Processor Time
    • Available MBytes
    • Disk Queue Length
    • Network Interface

    c) Monitor de recursos (resmon)

    • Interfície detallada sobre:
    • Activitat de CPU, memòria, disc i xarxa.
    • Relació entre processos i recursos consumits.

    d) Visor d’esdeveniments (eventvwr.msc)

    • A més de seguretat, registra errors i advertències del sistema, aplicacions i serveis.

    2.4 Solucions avançades i de tercers

    Per a entorns complexos o grans xarxes, és recomanable utilitzar eines més completes com:

    • Windows Admin Center: interfície centralitzada per gestionar i monitorar equips.
    • Nagios: eina de monitoratge de sistemes i xarxes.
    • Zabbix: monitoratge distribuït amb suport per a alertes i visualitzacions.
    • SolarWinds: solució comercial per a la monitorització integral d'infraestructures.

    2.5 Alertes i notificacions

    Es poden configurar alertes per detectar anomalies o fallades. Per exemple:

    • Configurar alertes en Performance Monitor per rebre una notificació si l’ús de CPU supera el 90% durant més de 5 minuts.
    • Utilitzar Task Scheduler per executar scripts automàtics davant de determinats esdeveniments.
